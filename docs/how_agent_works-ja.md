# 🔄 How It Works: 対話型統合エージェントループ (Interactive Agent Loop)

[English](how_agent_works-en.md) | [한국어](how_agent_works.md) | [日本語](how_agent_works-ja.md) | [简体中文](how_agent_works-zh.md)

本ドキュメントは、`generate-tizen-app`プロジェクトにおいて、**AIエージェント（Gemini、Claudeなど）と協業しながら段階的にTizenアプリを作成していく対話型ループ**の動作原理とアーキテクチャについて説明します。

1つのスクリプト実行でコードを生成するCLIモード（`how_cli_works.md`）とは異なり、エージェントループは絶えず文脈を維持し、ユーザーの要件を柔軟に受け入れ、即座にエラーを修正する「ペアプログラミング」方式です。

---

## 🏗️ 1. エージェンティック開発ループのサイクル

対話型の統合エージェント環境（例：Cursor、VS Code AI拡張機能、Antigravityなど）でこのリポジトリをワークスペースとして開き、次のようなサイクルを反復します。

### Step 1: 自然言語要件の受付 (Intent & Reasoning)
- ユーザーが「メッセンジャーの入力欄があるUIを作って」と指示します。
- エージェントはプロンプトを分析し、Tizenフレームワーク内でどの構成要素（Scaffold, Grid, TextFieldなど）が必要かを**事前に推論**します。

### Step 2: ツール(Tools)および知識の呼び出し (Context Gathering)
エージェントは単に内蔵の知識でコードを生成するのではなく、必要な仕様を即座に外部から調達します。
- **MCPサーバー連携 (`microsoft-learn`)**: TizenだけでなくC#の最新 .NET 8.0 構文や外部の .NET ライブラリが必要な場合、公式リファレンスを直接検索します。
- **ローカルスキル呼び出し (`tizen-api-search`)**: Tizen UIの `TextField` コンポーネントがパディング属性を持つか不明確な場合、ワークスペースにある `scripts/search-tizen-api.js` スクリプトをバックグラウンドで実行し、**実際の `api-index.json` 仕様書を確認**して、幻覚（Hallucination）なしに正確なプロパティ値を抽出します。

### Step 3: コード作成および修正 (Execution)
- 獲得した文脈と知識をもとに、`MainView.cs` などのコアコードを作成・修正します。
- 複数ファイル（Multi-file）の修正が必要な場合（例：App.csの修正、リソースの追加など）でも、エージェントが直接ファイルシステムにアクセスして編集します。

### Step 4: 検証および自己修復 (Verification & Self-Healing)
- エージェントが主導してターミナルコマンド（`dotnet build`）を実行し、コードが Tizen 10.0 / .NET 8.0 環境で実際にコンパイルできるかテストします。
- エラー（`CS1061`, `CS0246` など）が発生した場合：
  1. エージェントがターミナルのエラーログを読み、原因を自ら把握します。
  2. 「ああ、`Button`には`SetColumn`がなかったな。もう一度Tizen APIスキルを検索しよう」と動的に知識を補います。
  3. コードにパッチを当てて再度ビルドする **Self-Healing ループ**をエラーがなくなるまで繰り返し、ユーザーに完了を報告します。

## ⚙️ 2. なぜエージェントループなのか？ (CLIとの違い)

| 比較項目 | 対話型統合エージェントループ | スタンドアロンCLIツール (`Generate-App.js`) |
| --- | --- | --- |
| **コード生成方式** | 複数ツールの活用と段階的改善 | ワンショットに基づく全体コードブロックの生成 |
| **API知識の活用** | エージェントが能動的にスキル駆動・検索語を判断 | ツールがエラーを解析して機械的にヒントを注入 |
| **ファイル変更単位** | 自由度が高い（UI, ロジック, 設定などの同時変更可能） | 制限的（`MainView.cs`の置換が中心） |
| **ユースケース** | 複雑なアプリ開発、コードのリファクタリング、デバッグ | パイプライン自動化、テンプレートの抽出 |

## 🛡️ 3. 拡張性 (Extensibility)

エージェントループの最大の長所は、**ワークスペース(Workspace)自体がエージェントのツールボックス**になるという点です。
- **スキルの追加**: Tizen.UIだけでなく、`Tizen.Multimedia` や `Tizen.NUI` などの他のTizen APIのための検索スキルやドキュメントを `docs/` や `.agent/skills/` ディレクトリに追加するだけで、エージェントがすぐに読み取り、新たな開発能力を吸収します。
- **MCP連携**: 今後、Tizenアセンブリ DLLを直接分析するカスタム MCP（`dotnet-assembly-inspector` など）が連携されれば、JSON カタログすら不要で、DLL バイナリを直接解析してコーディングするという極限の自動化もサポートします。
